# 2026-03-21 工程规划

## 核心目标

- 新工程独立建仓，不和清洗工程混在一起
- issue 级拆解投诉问题
- 自动发现问题簇，不预设固定类别数
- 形成既能“本次发现问题”又能“长期复用”的双层类型体系

## 核心设计

- `WorkingType`：本次自动发现的问题簇
- `CanonicalType`：沉淀后的正式问题类型
- `IssueFacts`：不固定字段长度的结构化限定属性
- `CategoryDrilldown`：每个类别都能回查到具体工单和证据句

## 运行模式

- CPU 基线模式：允许在无 GPU 环境跑通完整链路
- GPU 增强模式：在更强机器上提升语义表示和聚类质量

## 关键原则

- 不要求先人工看完全部工单
- 不把类数写死
- 工程可跨年份、跨地区复用
- CLI、README、requirements、测试和文档一起交付
